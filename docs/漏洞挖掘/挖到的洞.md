JWT 即JSON Web Token

它由三部分组成（用`.`分隔）：

  * **Header（头部）** ：指定加密算法（如 HMAC SHA256）。
  * **Payload（载荷）** ：存储用户标识（如用户 ID）、过期时间等非敏感信息。
  * **Signature（签名）** ：服务器用密钥对前两部分加密生成的签名，用于验证令牌完整性。

可能会出现的JWT漏洞：

  * **空加密算法**



header = {

"typ": "JWT",

"alg": "HS512" → "alg": "none"

}

将加密方式改成None且不添加signature的内容，就会让服务器跳过对签名的验证从而直接信任JWT的内容，再配合一些修改管理员密码等接口就可以实现越权。

  


  * **弱密钥**



{

"claimList": { ——不重要

"roles": [

"admin",

"user"

],

"userid": "user" ——认证点

},

"exp": 1728784080, ——时间戳

"iat": 1728784080,

"sub": "1"

}

将密钥中的内容通过jwt.io改成"admin"就可以实现越权登陆，但一般都是单一认证因子的JWT才比较好用
