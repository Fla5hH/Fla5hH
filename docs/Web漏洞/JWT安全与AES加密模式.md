# 什么是JWT

**模式**| **类型**| **并行性**| **错误传播**| **推荐场景**  
---|---|---|---|---  
ECB| 基础模式| ✅| 无| 禁止使用（安全性极差）  
CBC| 基础模式| ❌| 全分组| legacy 系统（需搭配 HMAC）  
CTR| 流密码模式| ✅| 无| 高性能场景（需搭配 HMAC）  
CFB| 流密码模式| ❌| 多分组| 早期流数据（已被 CTR 替代）  
OFB| 流密码模式| ❌| 单分组| 低错误敏感度场景（少用）  
  
  * **CBC：Cipher Block Chaining** ，密码块链，明文被分成固定大小的块，并按顺序进行加密，每一个块（分组）要先和前一个分组加密后的数据进行 XOR 异或操作，然后再进行加密。 这样每个密文块依赖该块之前的所有明文块，为了保持每条消息都具有唯一性，第一个数据块进行加密之前需要用**初始化向量 IV** 进行异或操作。 CBC 模式是一种最常用的加密模式，它主要缺点是加密是连续的，不能并行处理，并且与 ECB 一样消息块必须填充到块大小的整倍数。



![](https://cdn.nlark.com/yuque/0/2025/png/51646461/1756496874503-d4eb1dcc-88bb-4429-8686-5e4d3c1ab052.png)

  * **ECB：Electronic Codebook（安全性低，不常用）** , 电子密码本，是最简单的加密模式，明文消息被分成固定大小的块（分组），并且每个块被单独加密。 每个块的加密和解密都是独立的，且使用相同的方法进行加密，所以可以进行并行计算，但是这种方法一旦有一个块被破解，使用相同的方法可以解密所有的明文数据，安全性比较差。 适用于数据较少的情形，加密前需要把明文数据填充到块大小的整倍数。



![](https://cdn.nlark.com/yuque/0/2025/png/51646461/1756497181435-6c3ac74f-4874-4e09-9391-b71e7aebc16b.png)

  * **GCM：Galois/Counter Mode（较主流）** ，伽罗瓦/计数器模式，是一种同时提供加密和认证的认证加密模式。其加密部分基于计数器模式（CTR）：明文被分成固定大小的块，通过加密不断递增的计数器生成伪随机流，与明文块异或得到密文块，支持并行处理。认证部分则通过伽罗瓦域乘法，对密文和关联数据（AD，无需加密但需认证的数据）计算认证标签（Tag）。每条消息需使用唯一的初始向量（IV），确保计数器不重复。GCM的优势是加密与认证可并行执行，且能同时保证机密性与完整性，缺点是对IV的唯一性要求严格，同一密钥下IV重复会导致安全漏洞。
